# PixivVault 機能一覧

現時点（2026-07-13）のコードベースを基に整理した機能一覧です。

## 1. 個別ダウンロード（タブ「個別DL」）
- PixivユーザーIDを指定して、そのユーザーの作品を一括ダウンロード（[gui.py](gui.py) タブ1 / [core.py](core.py) `run_backup`）
- 実行モード
  - 差分DL: DB記録と `update_date` を比較し、新規・更新・削除復帰した作品のみダウンロード
  - 完全DL: 全作品を強制的に再チェック
- 対象種別を選択可能（イラスト・漫画 / 小説 / 両方）
- 進捗バー・残り時間（ETA）表示
- 一時停止・再開・停止ボタン
- Pixiv側で削除された作品を検知し、DB上で削除フラグを立てて通知（`mark_as_deleted`）
- ダウンロード完了後、設定に応じて作者フォルダをZIP圧縮（新規ダウンロードがない回でも既存フォルダをZIP化可能）

## 2. フォロー中一括ダウンロード（タブ「フォロー中一括」）
- DBに保存済みの「フォロー中作者」一覧から複数選択して一括ダウンロード（[core.py](core.py) `run_batch_backup`）
- 全選択 / 全解除 / お気に入りのみ選択
- 名前検索、並び替え（フォロー登録順 / 名前順、昇順・降順）
- 対象種別（イラスト・漫画 / 小説 / 両方）の指定
- 一括進捗バー・作者単位の進捗（○/○人）・残り時間表示
- 一時停止・再開・停止
- 未フォロー作者バッジ（新着未フォロー数の表示、後述）

## 3. ブックマーク一括ダウンロード（タブ「ブックマーク」）
- ログイン中アカウントのブックマーク作品を一括ダウンロード（[core.py](core.py) `download_bookmarks`）
- 対象種別（イラスト・マンガ / 小説 / 両方）、公開範囲（公開 / 非公開）を指定可能
- 保存モード
  - フラット配置（direct）: ブックマーク専用フォルダ直下にまとめて保存
  - 階層配置（link）: 作者別フォルダを作成しハードリンクで整理
- 進捗・残り時間表示、一時停止・再開・停止
- **未フォロー作者を確認**: ブックマークはしているがフォローしていない作者を抽出し、代表作サムネイル付きダイアログで一覧表示。その場で「フォロー」ボタンから直接フォロー可能（[core.py](core.py) `get_unfollowed_bookmark_authors`）
- サムネイル画像はローカルキャッシュ（`cache/thumbnails`）に保存し再取得を削減

## 4. 小説保存機能
- 本文をEPUB / TXT / 両方の形式で保存可能（設定で選択）
- ルビ記法（`[[rb:漢字 > かんじ]]`）を各フォーマットに応じて変換
- 改ページタグ（`[newpage]`）の変換
- 挿絵・表紙画像の埋め込みダウンロードとEPUBへの組み込み

## 5. 拡張機能連携（ブラウザ拡張 + ローカルサーバー）
- Chrome/Edge向け拡張機能（[extension/](extension/)）
  - 作品ページ・小説ページに「PixivVaultに保存」ボタンを注入し、ワンクリックでダウンロード指示を送信
  - フォロー中一覧・ユーザーページに「差分DL」ボタンを注入
  - 保存済み状態（✅ 保存済 / ☑ 件数）をアイコン・カード上に表示
  - PixivVault本体が起動していない場合、カスタムURIスキーム（`pixivvault://start`）で自動起動し、起動待ちの間リトライ（最大7回）
  - fetchにタイムアウト（8秒）を設定し、応答なしでボタンが固まらないようにする
- ローカルHTTPサーバー（[server.py](server.py)、ポート25010）
  - `GET /api/work/{id}` 保存済み状態確認
  - `GET /api/user/{id}/status` ユーザーの保存済み件数確認
  - `POST /download` ユーザー/作品のダウンロードをキューに登録
  - Origin検証（`chrome-extension://` のみ許可）でローカルサーバーへの不正アクセスを防止
  - 同一ジョブの重複キュー登録を防止
  - ダウンロードキューのログをGUI「キュー」タブに転送
  - ダウンロード開始/完了/失敗をWindows通知（トースト）で通知
- GUI「拡張機能」タブ: レジストリへのカスタムURIスキーム登録・解除（自動起動設定のON/OFF）
- GUI「キュー」タブ: 拡張機能経由のダウンロード処理ログを表示

## 6. 定期自動チェック（スケジューラ）
- バックグラウンドスレッドで一定間隔ごとに「お気に入り」登録済み作者を自動チェック（[scheduler.py](scheduler.py)）
- チェック間隔を設定可能（6時間 / 12時間 / 24時間 / 48時間 / 無効）
- 新規保存があった場合はWindows通知で件数を知らせる

## 7. データ管理・設定（設定ダイアログ）
- Cookieインポート: `cookies.txt`（Netscape形式）をファイル選択でインポートし、ログイン状態を反映
- フォロー中リスト同期: Pixivから最新のフォロー中作者一覧を取得しDBへ保存
- 保存先フォルダー変更（ネイティブフォルダ選択ダイアログ）
- 小説の保存形式（EPUB/TXT/両方）
- お気に入り自動チェック間隔
- 新着通知のON/OFF
- ブックマーク保存モード（フラット/階層）
- 詳細設定（Advanced）
  - 全作者ダウンロード後自動ZIP化のON/OFF
  - 最小化時にタスクトレイへ格納するかどうか
  - サムネイルキャッシュの削除

## 8. エクスポート/バックアップ
- 作品メタデータ（DB内容）をJSONダンプ
- DBファイル（`pixiv_vault.db`）のコピー
- 画像フォルダ全体のコピー
- 上記をまとめてZIPアーカイブとして出力（[core.py](core.py) `export_data`）

## 9. ログイン状態チェック・Cookie監視・拡張機能自動同期機能 (#18)
- 起動時および定期バックグラウンド処理により `cookies.txt`（Netscape形式）の有効期限Unixタイムスタンプ・API疎通状態を検証
- **拡張機能連携によるCookie自動同期**: ブラウザ拡張機能がPixiv閲覧時・起動時に最新のCookieを自動取得し、ローカルサーバー（`POST /api/cookie/sync`）経由で `cookies.txt` を自動更新
- 自動同期成功時には、キューログ画面へ **マトリックス・サイバーグリーン（`#00FF66`）** でリアルタイム通知し、UIヘッダーのステータスバッジも即座に更新
- UIヘッダーにリアルタイムな状態バッジを表示
  - ● [有効] ログイン中 - ユーザーID
  - ▲ [期限切れ間近] 残り日数を警告
  - × [未認証/期限切れ] 再インポートを促す警告

## 9-2. ダウンロード失敗キュー＆保存品質自動チェック (#5, #6)
- **保存品質チェック (#6)**: ダウンロード直後に作品ファイルの整合性（0バイト検出、ZIP破損検証 `testzip()`, ページ数不足, 空ZIP）を自動検査
- **ダウンロード失敗キュー (#5)**: 通信エラー、404、品質チェック異常を検出した作品を処理を止めずに分離記録（`failed_queue` テーブル）
- **「失敗リスト」タブ**: 失敗理由・リトライ回数・ID付きの一覧表示、個別再試行ボタン、「全件再試行」「CSV出力」「全件クリア」ボタンを完備

## 10. タスクトレイ・ウィンドウ管理
- タスクトレイ常駐（[tray.py](tray.py)）: 「PixivVaultを開く」「終了」メニュー
- 最小化時にタスクトレイへ格納する設定
- ウィンドウを閉じる際、ダウンロード中であれば警告し、確認の上で全フローを停止してから終了
- 多重起動防止（ローカルポートを使った単一インスタンスロック）
- カスタムURIスキーム経由起動時のカレントディレクトリ自動補正

## 11. ログ表示
- メイン画面下部にリアルタイムログ表示（折りたたみ可能）
- 拡張機能経由のログは別タブ（「キュー」）に分離表示

## 使用している主な保存先ディレクトリ
- `Images/` : ダウンロードした作品の保存先（設定で変更可能）
- `☆ブックマーク(ユーザーID)/` : ブックマーク一括DL専用フォルダ
- `cache/thumbnails/` : 未フォロー作者確認機能のサムネイルキャッシュ
- `PixivVault_Backup_YYYYMMDD.zip` : エクスポート出力
